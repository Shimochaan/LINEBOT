# 🔮 おみくじENJINボット (LINE Messaging API ✕ Claude API ✕ MySQL ✕ PHP)

LINEで話しかけると、AI（Claude）が「親しみやすい占い師」として返信してくれるLINE Botアプリケーションです。
さらに今回のアップデートで、**「サウナ」「ランニング」などのアクティビティをLINEに送るとデータベースに記録し、「今月何回？」と聞くとSQLで集計して返す**機能を追加しました。
生年月日・性別・名前などの情報は会話の中で記憶し、星座・数秘術・九星気学などの占いを交えて前向きなアドバイスを返します。
さくらインターネット（レンタルサーバー）の環境特性を考慮し、セキュアかつ安定して稼働するバックエンドロジックをPHPで構築しました。

## 🛠️ 主な機能
1. **メッセージの意図判定（インテント振り分け）**
   * 届いたメッセージを `detectIntent()` で「集計」「サウナ記録」「ラン記録」「占い（それ以外）」に振り分けます。
   * **判定順序が重要**で、「今月サウナ何回？」のような *質問* を *記録* と誤認しないよう、**集計 → 記録** の順にチェックしています。
2. **アクティビティのデータベース記録（MySQL / PDO）**
   * 「サウナ」「走った」などのメッセージを受け取ると、`activity_logs` テーブルに1行 `INSERT` します。
   * SQLは必ず**プリペアドステートメント（`?` プレースホルダ）**で組み立て、SQLインジェクションを防いでいます。
3. **今月のアクティビティ集計（SQL集計）**
   * 「今月何回？」等の質問には、`GROUP BY` と `COUNT(*)` で当月分を集計し、サウナ・ランの回数を返信します。
4. **AI占い師との対話（会話履歴つき）**
   * 記録・集計以外のメッセージは Claude API（Anthropic, `claude-haiku-4-5`）に渡し、占い師キャラクターとして短文で返信します。
   * ユーザーごとの会話履歴を引き継ぐことで、一度教えた名前や生年月日を覚えたまま会話できます。
5. **署名検証によるセキュアなWebhook受信**
   * LINEからのリクエストを `X-Line-Signature`（HMAC-SHA256）で検証し、なりすましリクエストを拒否します。

## 💻 開発環境・使用技術
* **サーバー:** さくらインターネット（スタンダードプラン / CGIモード）
* **バックエンド:** PHP 8.x
* **データベース:** MySQL（PDO経由でアクセス）
* **外部API:** LINE Messaging API (Webhook) / Claude API（Anthropic, `claude-haiku-4-5`）
* **データ保存:** MySQL（アクティビティ記録）＋ サーバー上のJSONファイル（ユーザー別の会話履歴）

## ⚙️ セットアップ手順

このリポジトリには認証情報（実キー・DBパスワード）は含まれていません。
動かすには、雛形をコピーして自分のキーを設定する必要があります。

1. **設定ファイルを作成する**

   ```bash
   cp config.sample.php config.php
   ```

2. **`config.php` に自分のキー・接続情報を記入する**

   | 変数 | 取得場所・内容 |
   | --- | --- |
   | `$accessToken` | LINE Developers コンソール > Messaging API設定 > チャネルアクセストークン |
   | `$channelSecret` | LINE Developers コンソール > チャネル基本設定 > チャネルシークレット |
   | `$claudeApiKey` | Anthropic コンソール（ https://console.anthropic.com/ ）> API Keys |
   | `$errorLogPath` | エラーログを出力したいサーバー上の絶対パス |
   | `$historyDir` | 会話履歴JSONを保存するサーバー上のディレクトリの絶対パス |
   | `$db_host` / `$db_name` / `$db_user` / `$db_pass` | MySQLデータベースの接続情報 |
   | `$historyTurns` | AIに渡す会話履歴の往復数（既定 6往復） |

3. **サーバーに `index.php` と `config.php` を配置する**
   * `config.php` は `.gitignore` で除外されているため、サーバーには手動でアップロードします。
   * `activity_logs` テーブルは初回アクセス時に `CREATE TABLE IF NOT EXISTS` で自動作成されます。
   * `$historyDir` のディレクトリは初回アクセス時に自動作成されますが、書き込み権限が必要です。

4. **LINE側のWebhook URLに `index.php` のURLを設定する**

## 💡 技術的なこだわり・PHPの原理原則の適用

フロントエンド開発（JavaScript/React）の経験を活かしつつ、バックエンド特有の「サーバー環境の差異」や「セキュリティ」を意識して実装しました。

### 1. 意図判定の「順序」で誤記録を防ぐ
「今月サウナ何回行った？」という *質問* には「サウナ」という単語が含まれます。単純に上から単語一致で判定すると、質問なのにサウナを1回記録してしまいます。
そこで `detectIntent()` では、**まず集計・質問系のキーワード（「何回」「集計」等）を先に判定**し、その後に記録系（サウナ/ラン）を判定する順序にすることで、この取り違えを防いでいます。

### 2. SQLインジェクション対策（プリペアドステートメント）
ユーザーの入力（`userId`）をSQLに埋め込む際、文字列連結ではなく `prepare()` + `?` プレースホルダ + `execute([...])` を徹底しています。
これにより、悪意ある入力でSQL文が改ざんされる「SQLインジェクション」を構造的に防いでいます。

### 3. サーバー環境（CGIモード）に依存しないヘッダー取得
当初、署名検証用に `getallheaders()` を使用していましたが、さくらサーバーのCGIモードの特性で関数未定義エラー（503）を起こすことが判明しました。
環境依存を排除するため、`$_SERVER['HTTP_X_LINE_SIGNATURE']` から直接ヘッダーを取得する堅牢なコードに修正しています。

### 4. cURL通信のハングアップ防止（503エラー対策）
LINE・Claude APIへの通信（cURL）がハングアップしてサーバープロセスを逼迫させないよう、`CURLOPT_TIMEOUT` によるタイムアウト安全弁を実装しています。

### 5. 会話履歴のコンテキスト保持とファイル名の安全化
ステートレスなWebhookでも文脈のある会話ができるよう、ユーザーIDごとに会話履歴をJSONファイルへ永続化しています。
ファイル名にはユーザーIDを **SHA-256でハッシュ化**した値を使い（`historyFilePath()`）、生のIDが露出したり不正なパス操作が行われるのを防いでいます。保存時は `LOCK_EX` で排他ロックし、ディレクトリは `0700` 権限で作成します。また直近の往復数（既定6往復）に絞ることで、ファイル肥大化とAPIトークン消費を抑えています。

### 6. 認証情報の分離とGit管理からの除外
LINEの各トークン・Claude APIキー・DBパスワードなどの機密情報が生の文字列のままリポジトリに残らないよう、設定ファイル（`config.php`）に分離し `.gitignore` で除外しています。
公開リポジトリには雛形（`config.sample.php`）のみを含め、実キー・実パスワードは本番サーバー内の `config.php` にのみ配置する構成としました。
